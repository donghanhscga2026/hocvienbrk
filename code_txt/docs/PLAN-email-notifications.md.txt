# PLAN: Email Campaign Manager — Hệ Thống Gửi Email Thông Báo

## Mô tả tính năng

Xây dựng module **Email Campaign Manager** tích hợp vào Admin, cho phép admin:
1. **Quản lý danh sách email nguồn gửi** (1 main + nhiều satellite accounts)
2. **Lên kế hoạch chiến dịch** với tùy chọn nguồn người nhận & loại thông báo
3. **Draft → Activate → Run**: hệ thống tự chia batch gửi qua pool email
4. **Theo dõi deliverability**: phát hiện email bounce/không hợp lệ, báo cáo để admin xử lý
5. **Tuân thủ Gmail & CAN-SPAM**: mọi email có link hủy đăng ký, danh sách opt-out tập trung

> **Nền tảng có sẵn**: `lib/notifications.ts → sendGmail()` via Gmail OAuth2.  
> **Chiến lược scale**: ~1000 học viên chia cho ~10 email vệ tinh, mỗi email gửi <500/ngày.

---

## User Review Required

> [!IMPORTANT]
> **Schema DB mới**: Thêm 3 model: `EmailSender`, `EmailCampaign`, `EmailCampaignLog`. Cần chạy `prisma migrate`. Xác nhận OK trước khi implement.

> [!IMPORTANT]
> **Mỗi email vệ tinh cần OAuth2 riêng**: Bạn sẽ cần lấy `refresh_token` cho từng email vệ tinh qua Google OAuth2 playground. Hệ thống sẽ có trang admin để khai báo chúng.

> [!WARNING]
> **Bảo mật refresh_token**: Các token của email vệ tinh sẽ được lưu trong DB (bảng `EmailSender`), được mã hóa bằng `AES-256`. Cần thêm biến `EMAIL_ENCRYPTION_KEY` vào `.env`.

---

## Kiến trúc tổng thể

```
Admin: Khai báo Email Pool (main + satellites)
           ↓
Admin: Tạo Campaign Builder (4 bước)
    1. Loại thông báo
    2. Nguồn người nhận  
    3. Soạn email
    4. Review & Activate
           ↓
/api/admin/campaigns/[id]/run
           ↓
Campaign Engine:
    - resolveRecipients() → danh sách email nhận
    - chunkBySender()     → chia batch theo email pool
    - runBatch(sender, recipients) → gửi từng batch
           ↓
sendGmailFromSender(senderAccount, to, subject, html)
           ↓
Gmail API (mỗi sender dùng refresh_token riêng)
```

## Proposed Changes

### Component 1 — Database Schema

#### [MODIFY] prisma/schema.prisma

**Model `EmailSender`** — Quản lý pool email nguồn gửi:
```prisma
model EmailSender {
  id            Int      @id @default(autoincrement())
  label         String                    // Tên gợi nhớ (VD: "Vệ tinh 01")
  email         String   @unique
  isMain        Boolean  @default(false)  // Email chính hocvienbrk@gmail.com
  clientId      String                    // GMAIL_CLIENT_ID
  clientSecret  String                    // Mã hóa AES-256
  refreshToken  String   @db.Text         // Mã hóa AES-256
  dailyLimit    Int      @default(480)    // Giới hạn gửi /ngày (an toàn dưới 500)
  sentToday     Int      @default(0)      // Đếm số đã gửi hôm nay
  lastResetAt   DateTime @default(now())  // Ngày reset sentToday
  isActive      Boolean  @default(true)
  createdAt     DateTime @default(now())
  campaigns     EmailCampaignSender[]
}
```

**Model `EmailCampaign`** — Chiến dịch email:
```prisma
model EmailCampaign {
  id                Int            @id @default(autoincrement())
  title             String
  status            CampaignStatus @default(DRAFT)
  notificationType  String         // NEW_LESSON | ZOOM_SESSION | NEW_POST | ANNOUNCEMENT | CUSTOM
  linkedEntityId    String?        // ID bài học / bài đăng liên quan
  recipientSource   String         // DB_ALL | DB_ACTIVE | DB_COURSE | CSV
  recipientFilter   Json?          // { courseId } nếu lọc theo khóa
  recipientCsvData  String?        @db.Text
  subject           String
  htmlContent       String         @db.Text
  totalRecipients   Int            @default(0)
  sentCount         Int            @default(0)
  failedCount       Int            @default(0)
  startedAt         DateTime?
  completedAt       DateTime?
  createdAt         DateTime       @default(now())
  updatedAt         DateTime       @updatedAt
  createdBy         Int
  creator           User           @relation(fields: [createdBy], references: [id])
  senders           EmailCampaignSender[]
  logs              EmailCampaignLog[]
}

model EmailCampaignSender {
  id          Int           @id @default(autoincrement())
  campaignId  Int
  senderId    Int
  assignedCount Int         // Số email được phân công cho sender này
  sentCount   Int           @default(0)
  campaign    EmailCampaign @relation(fields: [campaignId], references: [id], onDelete: Cascade)
  sender      EmailSender   @relation(fields: [senderId], references: [id])
}

model EmailCampaignLog {
  id          Int      @id @default(autoincrement())
  campaignId  Int
  senderId    Int?     // Gửi từ email nào
  toEmail     String
  status      String   // SENT | FAILED | SKIPPED | BOUNCED
  errorCode   String?  // VD: "5.1.1 user unknown", "550 mailbox full"
  errorType   String?  // HARD_BOUNCE | SOFT_BOUNCE | INVALID | TIMEOUT
  sentAt      DateTime @default(now())
  campaign    EmailCampaign @relation(fields: [campaignId], references: [id], onDelete: Cascade)
}

// Danh sách email không hợp lệ / opt-out — dùng chung toàn hệ thống
model EmailBlacklist {
  id          Int      @id @default(autoincrement())
  email       String   @unique
  reason      String   // HARD_BOUNCE | UNSUBSCRIBED | MANUAL
  campaignId  Int?     // Campaign phát hiện ra (nếu có)
  createdAt   DateTime @default(now())
}

enum CampaignStatus { DRAFT ACTIVE RUNNING COMPLETED FAILED }
```

### Component 5 — Email Templates & Compliance

#### [NEW] lib/email-templates/index.ts
- `newLessonTemplate(data)` — template bài học mới
- `zoomSessionTemplate(data)` — template buổi Zoom
- `newPostTemplate(data)` — template tin tức
- `announcementTemplate(data)` — template thông báo quan trọng

**Mọi template BẮT BUỘC có:**
- Header: Logo Học Viện BRK + tên người nhận
- Footer chuẩn Gmail/CAN-SPAM:
  ```
  Bạn nhận email này vì là học viên Học Viện BRK.
  [Hủy đăng ký nhận email] — link dẫn đến /api/unsubscribe?token=xxx
  ```
- Trường `List-Unsubscribe` trong email header (Gmail dùng để hiện nút Unsubscribe gốc)
- Không dùng spam trigger words: "Miễn phí!!!", "CLICK NGAY", quá nhiều chữ in hoa

#### [NEW] lib/email-spin.ts — Content Spinning Engine
**Mục đích:** Tạo nội dung biến thể khác nhau cho từng học viên, giảm trùng lặp → cải thiện deliverability.

**Cú pháp soạn thảo:**
```
{Chào bạn|Xin chào|Hi} [Tên],

Khóa học {đã được cập nhật|có bài học mới|vừa ra bài mới}!
Hãy vào xem {ngay|ngay hôm nay|sớm nhé}.
```
→ Mỗi học viên nhận 1 tổ hợp ngẫu nhiên, không ai giống ai.

**Hàm xử lý:**
- `spinContent(template, seed?)` — parse tất cả `{a|b|c}` trong chuỗi, random 1 lựa chọn
- `previewSpin(template, count?)` — tạo N bản preview để admin xem trước
- Hỗ trợ nested (tùy chọn Phase 2)

**Tích hợp trong Campaign Runner:**
- Gọi `spinContent(htmlContent)` cho mỗi email trước khi gửi
- Biến cá nhân hóa `[Tên]`, `[MãHV]` được replace sau khi spin

**Tích hợp trong UI (Bước 3 — Soạn email):**
- Hộp soạn thảo có nút `{a|b|c}` để chèn spin syntax
- Nút **[Preview 3 phiên bản]** — hiển thị 3 bản render mẫu khác nhau để admin kiểm tra

#### [NEW] app/api/unsubscribe/route.ts
- `GET /api/unsubscribe?token=<jwt>` — giải mã token → thêm email vào `EmailBlacklist` với lý do `UNSUBSCRIBED` → hiển thị trang xác nhận
- Token dùng JWT chứa `{ email, userId }` (không expire để link hoạt động mãi)

### Component 8 — Bounce Tracking & Deliverability Report

#### [MODIFY] lib/email-campaign-runner.ts
Bổ sung xử lý lỗi Gmail API:
- Phân loại lỗi gửi:
  - **Hard Bounce** (5xx): email không tồn tại, domain không hợp lệ → tự động thêm vào `EmailBlacklist`
  - **Soft Bounce** (4xx): hộp thư đầy, server tạm thời lỗi → ghi log, cho phép retry
  - **SKIPPED**: email đã có trong `EmailBlacklist` → không gửi, ghi log
- Trước khi gửi từng email: kiểm tra `EmailBlacklist` → skip nếu có

#### [NEW] app/api/admin/campaigns/[id]/bounce-report/route.ts
- `GET` — trả về danh sách email FAILED/BOUNCED của campaign, phân loại theo `errorType`

### Chiến lược Anti-Spam (Áp dụng toàn bộ)

| Biện pháp | Chi tiết |
|-----------|----------|
| **Unsubscribe link** | BẮT BUỘC trong mọi email (Gmail policy 2024) |
| **List-Unsubscribe header** | Gmail hiển thị nút unsubscribe gốc |
| **Personalization** | Luôn chứa tên học viên (`Chào [Tên],`) |
| **Content Spinning** | `{a|b|c}` — mỗi email nhận nội dung biến thể khác nhau |
| **Sender reputation** | Gửi chậm 200ms/email, không spam burst |
| **Email validation** | Kiểm tra format email trước khi gửi |
| **Blacklist pre-check** | Skip email đã bounce hoặc unsubscribe |
| **HTML/Text ratio** | Template có đủ text, không chỉ là ảnh |
| **Subject line** | Không ALL CAPS, không quá nhiều dấu `!`, có tên tổ chức `[Học Viện BRK]` |
| **Consistent sender** | Luôn hiển thị `Học Viện BRK <hocvienbrk@gmail.com>` trong From |

### Manual Tests (Step-by-step)

**Test 1 — Khai báo email vệ tinh:**
1. Mở `/admin/email-senders` → Click "Thêm email vệ tinh"

4. Xác nhận: sender xuất hiện trong bảng với status Active

**Test 2 — Tạo campaign DRAFT:**
1. Mở `/admin/campaigns/new`
2. Bước 1: Chọn "Thông báo quan trọng"
3. Bước 2: Chọn "Upload CSV" → upload file 2-3 email test
4. Click "Preview danh sách" → xác nhận đúng số người nhận
5. Bước 3: Nhập tiêu đề + nội dung
6. Bước 4: Click "Lưu nháp" → kiểm tra DB: `EmailCampaign.status = DRAFT`

**Test 3 — Gửi thử (CSV nhỏ):**
1. Mở campaign DRAFT vừa tạo
2. Click "Kích hoạt & Gửi"
3. Xác nhận: status chuyển RUNNING → COMPLETED
4. Kiểm tra inbox các email trong CSV nhận được email
5. Xem log: `sentCount` đúng, `failedCount = 0`

**Test 4 — Phân bổ multi-sender:**
1. Khai báo 2 senders (1 main + 1 vệ tinh)
2. Tạo campaign với CSV chứa 10 email
3. Click "Xem phân bổ sender" → xác nhận hệ thống chia đều
4. Chạy campaign → kiểm tra log: messages được gửi từ 2 sender khác nhau
