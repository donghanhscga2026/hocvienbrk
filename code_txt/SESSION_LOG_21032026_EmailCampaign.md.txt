# SESSION LOG: Triển Khai Email Campaign Manager
**Ngày**: 21/03/2026
**Dự án**: HocVien-BRK
**Agent**: Gemini CLI (@project-planner, @frontend-specialist, @documentation-writer)

---

## 1. MỤC TIÊU PHIÊN LÀM VIỆC
Xây dựng module **Email Campaign Manager** tích hợp vào hệ thống Admin hiện tại, cho phép gửi email thông báo hàng loạt qua nhiều tài khoản vệ tinh (Pool Senders) để vượt qua giới hạn của Gmail và Vercel Serverless.

---

## 2. CÁC THỐNG NHẤT QUAN TRỌNG (SOCRATIC GATE)

Trong phiên làm việc, chúng ta đã thống nhất các phương án tối ưu sau:
- **Xử lý Timeout Vercel Free (10s)**: Sử dụng phương án "Client-Side Batching". UI phía Admin sẽ gọi API gửi từng đợt nhỏ (Batch) và lặp lại cho đến khi hoàn thành.
- **Kết nối Google Account**: Sử dụng luồng OAuth2 tự động. Admin chỉ cần nhấn "Kết nối", hệ thống tự lấy Email và Refresh Token, mã hóa và lưu vào DB.
- **Bảo mật**: Dùng AES-256-CBC với key 64 ký tự hex để mã hóa các thông tin nhạy cảm (Refresh Token, Client Secret).
- **Content Spinning**: Hệ thống xử lý Spin ngay lúc gửi để mỗi email gửi đi là một bản biến thể khác nhau, tăng uy tín cho tài khoản gửi.

---

## 3. CÁC THÀNH PHẦN ĐÃ TRIỂN KHAI

### 3.1. Database & Security
- **Schema**: Thêm các bảng `EmailSender`, `EmailCampaign`, `EmailCampaignSender`, `EmailCampaignLog`, `EmailBlacklist`.
- **Encryption**: File `lib/email-encryptor.ts` xử lý mã hóa AES-256.

### 3.2. Core Logic (Email Engine)
- **Google Auth**: File `lib/google-auth.ts` quản lý OAuth2 Client.
- **Spinning**: File `lib/email-spin.ts` xử lý cú pháp `{Chào|Hi}`.
- **Campaign Runner**: File `lib/email-campaign-runner.ts` điều phối việc lấy người nhận và tạo MIME email chuẩn Gmail API.

### 3.3. API Routes
- **/api/admin/auth/google**: Khởi tạo luồng OAuth.
- **/api/admin/auth/google/callback**: Nhận code, lưu sender.
- **/api/admin/campaigns**: Quản lý chiến dịch (Tạo/Lấy chi tiết).
- **/api/admin/campaigns/[id]/send-batch**: API "xương sống" xử lý gửi từng đợt email.
- **/api/unsubscribe**: API giúp học viên hủy đăng ký (Compliance).
- **/api/cron/reset-sender-quota**: Reset quota gửi về 0 mỗi ngày.

### 3.4. Admin Interface
- **Sidebar**: Thêm 2 mục menu mới.
- **Email Pool Manager**: Giao diện quản lý các tài khoản vệ tinh.
- **Campaign Builder**: Form 4 bước soạn thảo chiến dịch chuyên nghiệp.
- **Monitoring Page**: Trang theo dõi tiến độ gửi với Progress Bar thời gian thực.

## 4. HƯỚNG DẪN KÍCH HOẠT (BẮT BUỘC)

Hệ thống đã được code xong 100%, bạn chỉ cần thực hiện 2 bước cuối:

1. **Cập nhật Database**:
   ```bash
   npx prisma db push
   ```

2. **Cấu hình .env**:
   - `EMAIL_ENCRYPTION_KEY`: Chạy `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` để lấy key.
   - `GMAIL_CLIENT_ID`: Lấy từ Google Cloud Console.
   - `GMAIL_CLIENT_SECRET`: Lấy từ Google Cloud Console.
   - `GMAIL_REDIRECT_URI`: `http://localhost:3000/api/admin/auth/google/callback`
